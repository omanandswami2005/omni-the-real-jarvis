## WebSocket Message Schema

See `backend/app/models/ws_messages.py` for the full Pydantic schema definitions.

### Message Types (Client → Server)
- `auth` — Firebase ID token for session authentication
- `text` — Text message input
- `control` — Start/stop/interrupt session
- `image` — Image data (base64)
- `mic_acquire` — Request the voice mic floor before streaming audio (`/ws/live` only)
- `mic_release` — Release the mic floor when recording stops (`/ws/live` only)
- `tool_result` — T3 reverse-RPC response (client returning a local tool result)
- `capability_update` — Push updated capabilities/tools mid-session

### Message Types (Server → Client)
- `response` — Agent text response
- `transcription` — Real-time speech transcript
- `status` — Agent state changes (listening, thinking, speaking, tool_use)
- `tool_call` / `tool_response` — Tool invocation details and results
- `image_response` — Generated image with `image_base64` (live) and `image_url` signed URL (persistent)
- `cross_client` — Cross-client action dispatch; carries `cross_client: true` and `_origin_client_type`
- `mic_floor` — Mic floor state change; `event` is one of `granted`, `denied`, `acquired`, `released`, `busy`
- `client_status_update` — List of currently connected clients changed
- `session_suggestion` — Another device has an active session the user may want to join
- `error` — Error message

### Mic Floor Protocol (`/ws/live` only)

Only one client per user may stream audio to the ADK live session at a time.

```
Client A:  mic_acquire ──────────────────────────────► Server
Server:    mic_floor{granted, holder:"web"} ──────────► Client A   (unicast)
Server:    mic_floor{acquired, holder:"web"} ─────────► ALL clients (broadcast)
Client A:  <binary PCM frames> ──────────────────────► Server
Client B:  mic_acquire ──────────────────────────────► Server
Server:    mic_floor{denied, holder:"web"} ───────────► Client B   (unicast, rejected)
Client A:  mic_release ──────────────────────────────► Server
Server:    mic_floor{released, holder:"web"} ─────────► ALL clients (broadcast)
Client B:  mic_acquire ──────────────────────────────► Server  (now succeeds)
```

The server automatically expires a lock after **30 seconds of silence** from the holder, preventing indefinite blocking if a client crashes without releasing.

### Image Persistence

Images generated by `generate_image` / `generate_rich_image` tools are stored in GCS. During a live session `image_base64` is included in the `image_response` for immediate display. When loading session history from `GET /sessions/{id}/messages`, `image_base64` is absent but `image_url` (signed HTTPS, 60-min TTL) and `parts[].image_url` are populated so images survive page reloads.

```json
// Live session
{
  "type": "image_response",
  "tool_name": "generate_image",
  "image_base64": "iVBORw...",
  "image_url": "https://storage.googleapis.com/...",
  "mime_type": "image/png",
  "description": "A sunset over Tokyo"
}

// History (image_base64 absent, image_url always present)
{
  "type": "image",
  "image_url": "https://storage.googleapis.com/...",
  "mime_type": "image/png",
  "description": "A sunset over Tokyo"
}
```
