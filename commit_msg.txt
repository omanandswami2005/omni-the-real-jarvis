🧹 [code health improvement] Refactor `jules_create_session` to use config object

🎯 **What:** The `jules_create_session` function in `backend/app/plugins/jules_plugin.py` was taking too many parameters (`prompt`, `source`, `branch`, `title`, `auto_create_pr`). This PR introduces a new Pydantic `JulesSessionConfig` model to group these parameters into a single configuration object.

💡 **Why:** This improves maintainability and readability by reducing the parameter list and logically grouping session configuration. Moving parameter descriptions from docstrings to Pydantic `Field` descriptions also ensures the LLM retains schema context.

✅ **Verification:**
- Ran `ruff check` and `ruff format` to ensure no linting/formatting regressions.
- Checked test suite passes (outside of pre-existing unrelated environment failures).
- Code review gave a #Correct# rating.

✨ **Result:** The function now has a cleaner signature and is easier to extend in the future without adding more positional or keyword arguments.
